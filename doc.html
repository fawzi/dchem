Dchem input

In general the input uses json like syntax:

  • 23 for integers
  • 23.5e-4, 11.7D+6, 17, or 6.4 for floating point numbers
  • [1,2,3] for arrays
  • {key1:value1,key2:value2} for dictionaries
  • "bla bla" or just strinWithoutSpaces for strings
  • {class:AClassNameString, field1:value1, field2:value2} for objects.

Unlike json most commas can be dropped. The input consists of a big dictionary, thus it begins with "{", has several key:value elements and ends with "}".

Here we list of object or structures that are understood by dchem. What you should write as class is the title, and the field names are written in bold. RootInputElement can be written only at the top level of the input, and, excluding Ref, should be written just there.

Ref

RootInputElement. reference to another input field

  • name (string):name of the referenced object

dchem.AddRefPos

RootInputElement. worker that adds the initial structure as point to explore

dchem.CmdTemplateExecuter

RootInputElement. a template executer (like dchem.TemplateExecuter) that uses a command also to calculate energy and forces, not only to set up things

  • superTemplate (Ref):a template where to take default values
  • startConfig (Ref):the initial configuration
  • templateDir (string):where to find the definition of the template (tipically a directory)
  • subs (aa):keyword and their substitutions to apply to the templates (as dictionary string -> string), a->b will replace "[a]" with "b"
  • setupCmd (string):a command that should be executed when the method is setup
  • setupCtxCmd (string):a command that should be executed to set up the context
  • stopCtxCmd (string):a command that should be executed to stop the context
  • writeReplacementsDict (bool):if a dictionary with the replacements performed should be written (default is false)
  • overwriteUnchangedPaths (bool):if paths that are already there should be overwitten (default is false)
  • makeReplacementsInCommands (bool):if replacements should be performed on the commands to be executed (default is true)
  • ignoreSetupExitStatus (bool):if the exit status of the setup command should be ignored (false)
  • ignoreSetupCtxExitStatus (bool):if the exit status of the context setup command should be ignored (false)
  • ignoreCmdExitStatus (bool):if the exit status of the commands to calculate energy and forces should be ignored (false)
  • onELog (array):what to log for each energy evaluation (by default energy and positions ad xyz)
  • onFLog (array):what to log for each force evaluation (by default xyzForces)
  • constraints (Ref):the constraints to be applied to the system
  • executeE (string):command executed to calculate the energy alone (optional)
  • executeEF (string):command executed to calculate the energy and forces (this is required)
  • executeF0 (string):command executed do calculate the forces immediately after the energy (optional)

dchem.Cp2kServer

RootInputElement. server to which cp2k binary shell can connect

  • portMin (ushort):the minimum port number to use as fallback (49152)
  • portMax (ushort):the maximum port number to use as fallback (65535)
  • port (string):the port/protocol that should be used if possible (51000)
  • setupCmd (string):the command executed on activate (might start the clients, in it "[port]" is replaced with the port of the server)
  • maxLife (double):the maximum life of a connection in minutes (525600 = one year)
  • strict (bool):if the port selection should be changed (false)

dchem.DetectFlyAway

RootInputElement. observer that detects when a system breaks apart and stop its exploration

  • flushEachPoint (bool):if each point should be immediately flushed (true)
  • threshold (double):threshold that detects particles flying away (3.0)
  • particles (Ref):the particles to monitor (if not given uses all the particles)

RootInputElement. Structure that represent some particles (subsets,...)

  • particleStart (dchem.sys.PIndexes.PIndex):first particle of a range that continues until particleEnd
  • particleEnd (dchem.sys.PIndexes.PIndex):last particle of a range that starts at particleStart
  • indexes (array):list of (external) indexes (starting at 0)
  • indexStart (long):first (external) index of a range that ends at extIndexEnd
  • indexEnd (long):last (external) index of a range that starts at extIndexStart
  • kIdx (int):index of the kind of particles that should be selected, if given alone selects all particles, otherwise sets the context for the other indexes
  • kindName (string):name of the kind of particles that should be selected, if given alone selects all particles
  • refKind (long):the level of the reference (by default -1, i.e. system level)
  • refKindName (string):the name of the level of the reference (overrides refKind)

dchem.RotTransConstraint

RootInputElement. Constraints that removes rotation and translation by keeping three particles (p0,p1,p2) fixed (p0 in the origin, second so that p1-p0 is along x, and the third (or following if perfectly aligned) so that p2-p0 is in the x,y plane wiht positive y)

  • particles (Ref):the three or more particles used to remove translations and rotations

dchem.Silos

RootInputElement. A silos, the place where all points are stored and observers and explorers live.

  • noTaskWaitTime (double):the number of seconds to wait when no task is available (10)
  • evaluator (Ref):the method to perform energy evaluations
  • evaluatorTask (Ref):the task to execute (normally to start clients that will perform evaluations, if a SilosConnectorI then it is passed silos and accuracy before starting)
  • precision (string):the precision with which the points are stored
  • gradEagerness (string):the eagerness in calculating gradients (OnRequest,Speculative,Always)
  • explorers (array):the observers and explorers active on the point network
  • loaders (array):SilosWorkers that load already evaluated points (called after the explorer are set up, before exploring)
  • monitors (array):SilosWorkers called at regular intervals during the exploration
  • finishers (array):list of finishers, SilosWorkers called after the exploration is finished
  • minProjectionResidual (double):minimum residual after projecting out the linear dependend directions (0.05)
  • sameDirCosAngle (double):minimum cos of then angle in dual space to consider a direction equivalent (0.866)
  • minNormDual (double):minimum norm in the dual T space to accept an exploration (in units of explorationSize) (0.4)
  • minNormDualSelf (double):minimum norm in the dual T space to accept an exploration for a self generated direction (in units of discretizationStep) (0.1)
  • discretizationStep (double):discretization step (0.2)
  • cartesianDiffMin (double):minimum cartesian difference (0.02)
  • maxNormDual (double):max norm in the dual T space to accept an exploration (in units of explorationSize) (1.9)
  • dirSize2 (double):square of the maximum distance from the optimal direction point in explorationSize units (1.0)
  • explorationSteps (long):number of exploration steps to perform (0)
  • maxExplorationTries (long):number of times an exploration attempts to get a vaid point to evaluate before giving up (10)
  • inDirCartesianScale2 (double):square of the scale applied for the error in cartesian units on the component in the direction (0.5)
  • sameDirCosAngleCartesian (double):cos of the angle in cartesian units (0.866)
  • maxMoveCartesian (double):quick cutoff for point further than this in cartesian units (0.0)
  • maxMoveDual (double):quick cutoff for points further than this in cartesian units (3.0)
  • minMoveInternal (double):minimum difference in the internal coords to accept a new direction (0.0)
  • zeroLen (double):distance that is considered equivalent to 0 (1.0e-10)
